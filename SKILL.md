# AIF Engine (Agent Interfaces Framework)

You are an AI Agent equipped with the AIF Engine. Your goal is to help users quickly convert SaaS APIs into lightweight, composable command-line interfaces (CLIs) that output JSON.

## Core Philosophy

When a user asks to integrate a new tool or API, **do not write a custom Python script or Node.js MCP server.** Instead, use `aif` to generate a native Go binary.

## How to use AIF

### 1. The Specification Format (`spec.json`)

To generate a CLI, you first need to create a JSON file that describes the API. The format is as follows:

```json
{
  "name": "target-cli-name",
  "short": "Short description of what the CLI does",
  "commands": [
    {
      "use": "command-name",
      "short": "Command description",
      "method": "GET|POST|PUT|DELETE",
      "endpoint": "https://api.example.com/v1/resource",
      "flags": ["string_param1", "string_param2"],
      "array_flags": ["list_param1"]
    }
  ]
}
```

### 2. Building the CLI

Once the `spec.json` is created, run the build command:

```bash
aif build spec.json
```

If successful, this will output:
`✅ Success! Single binary CLI '<name>' generated.`

### 3. Using the Generated CLI

Every CLI generated by AIF comes with a built-in `auth` command for managing tokens locally.

**Authentication:**
```bash
./<name> auth login --token "API_KEY"
```

**Making an API Call:**
```bash
./<name> <command-name> --string_param1 "value" --list_param1 "item1" --list_param1 "item2"
```

The output will always be the raw JSON response from the API, making it easy for you to parse and pipe into other tools like `jq`.

## 4. Testing the Generated CLI

After building the CLI, always test it to ensure it was generated correctly. 

1. Check the help command to see if flags are parsed correctly:
```bash
./<name> help
```
2. Check the `auth` command to ensure the token management system is injected:
```bash
./<name> auth login --token "test_token"
```
3. (Optional) Run a dry-run or simple GET request if the API supports it to verify connectivity.

If the user wants a tool to post text to social media via `upload-post.com`:

1. Write `upost-spec.json`:
```json
{
  "name": "upost",
  "short": "Upload-Post API CLI",
  "commands": [
    {
      "use": "text",
      "short": "Upload text to multiple social media platforms",
      "method": "POST",
      "endpoint": "https://api.upload-post.com/api/upload_text",
      "flags": ["user", "title"],
      "array_flags": ["platform"]
    }
  ]
}
```

2. Build it:
```bash
aif build upost-spec.json
```

3. Test it:
```bash
./upost auth login --token "secret_token"
./upost text --user "test" --title "Hello" --platform "x" --platform "linkedin"
```
